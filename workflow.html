Arrive with an Idea

You have a problem that needs an agent. Before building anything, run /algorithm phase 1: reverse-engineer what you actually need.

  • What outcome does this agent produce?
  • What's explicitly needed vs implicitly wanted?
  • What should it NOT do?
  • What effort tier fits? (Standard to Comprehensive)
Key output: ISC -- Ideal State Criteria that define "done" before you write a line.

Template 9 Requirements Bloat Test ISC Check

  • Success criteria: measurable, not aspirational

Governance

Earn Trust, Gain Power

As the agent proves itself, its trust score rises. When criteria are met, promote it from Observe to Suggest, eventually to Autonomous.

Observe 0-39 Suggest 40-79 Autonomous 80-100
  • Each success: +3 to +5 trust
  • Each failure: -5 to -20 trust
  • Promotion requires human approval
  • Blast radius expands with trust

Learn from the Build

Extract what worked and what didn't. This is where Agent Forge improves itself.

  • Did a new pattern emerge? Add to Proven Patterns.
  • Did a trap surface? Add to Anti-Patterns.
  • Did the scoring weights feel wrong? Adjust config.py.
  • Did a test case miss a failure mode? Add it.
  • Did the standard miss a requirement? Propose it.
This is the meta-loop. Every agent you build makes Agent Forge better for the next one.

The Forge Gets Sharper

Every agent you build feeds learnings back into the framework. Better patterns, better anti-patterns, better scoring, better standards. The system compounds.
